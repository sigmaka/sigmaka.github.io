DBeaver#


← Einführung/Inhalt

DBeaver ist …

Lernziele#

  • Sie sind geübt im Umgang mit dem Tool DBeaver

  • Sie wissen, wie Sie Abfragen an MySQL formulieren müssen

1. Motivation#

Auf die Daten in einer Datenbank kann nicht einfach von aussen zugegriffen werden. Das Data Base Management System (DBMS) lässt das nicht zu. Anfragen müssen über eine Schnittstelle in SQL gestellt werden. Dies ist über die Eingabeaufforderung (Kommandozeile) zwar möglich, aber wenig komfortabel.

Das Programm DBeaver ist ein Datenbank-Administrations- und Analyse-Tool. Es vereinfacht die Verwaltung unserer Datenbanken. Im Moment gibt es davon eine Community-Version, die wir für den Unterricht einsetzen dürfen (nicht-kommerzielle Anwendung).

Es gibt andere Tools für die Verwaltung von DB. Bekannt sind

  • HeidiSQL: Open Source aber nur für Windows, kann keine Diagramme der Datenbank erzeu-gen

  • Oracles SQLWorkbench: Lange Zeit der Platzhirsch. Ist in die Jahre gekommen und unterstützt unser DBMS (MariaDB) nicht zu hundert Prozent.

2. Installation#

2.1 Laptop#

  1. Laden Sie die DBeaver Community Edition von https://dbeaver.io/ herunter und installieren Sie diese.

  2. Sie erhalten u. U. den Hinweis, dass der MariaDB-Treiber nicht installiert ist. Lassen Sie diesen installieren, denn wir verwenden die MariaDB.

  3. Sie werden gefragt, ob Sie eine Test-Datenbank anlegen möchten. Dies ist zu empfehlen, damit Sie unabhängig von anderen Komponenten und Software das Programm testen und nutzen können.

2.2 BLP#

Die BLP hat bereits das DBMS MariaDB installiert. Sie sollten aus dem vierten Semester bereits das aktuelle Image auf dem RPi installiert haben. Falls dies nicht der Fall ist, müssten Sie dieses nun nachholen. Die Anleitung finden Sie auf dem Infopoint:

Begleitdokumente Unterricht -> BLP_Basislernplattform -> 2022 -> V2

3. Workshop#

Im Theorieteil haben Sie die Abfragesprache SQL (Structured Query Language) kennengelernt. Damit werden nicht nur Daten aus der Datenbank abgefragt, sondern auch neue Datensätze angelegt und gelöscht. Damit die Befehle, die Sie kennengelernt haben, nicht nur Theorie bleiben, wollen wir uns im folgenden Workshop damit beschäftigen.

Vorbereitung Für diesen Workshop verwenden wir die Übungsdatenbank “employee”. Die Beschreibung finden Sie auf datacharmer/test_db. Die Datenbank ist bereits auf der BLP installiert. Daher brauchen Sie nichts zu tun.

Führen Sie die folgenden Schritte aus, um mit der Datenbank zu verbinden.

  1. Installieren sie DBeaver nach Anleitung.

  2. Starten Sie die BLP und stellen Sie sicher, dass sie sich im gleichen WLAN angemeldet hat, wie Ihr Computer.

  3. Erstellen Sie in DBeaver einen neuen MySQL-Connector:

  4. Folgende Settings: IP: 192.168.0.31 (oder WIFI-IP von Display) Port: 3306 User: pi Password: pi

  5. Verbindung testen (unten links), wenn gut

  6. Fertigstellen. Die Datenbank sollte im Datenbanknavigator erscheinen.

Aufträge Wir werden einige SQL-Befehle kennenlernen. Eine gute Übersicht finden Sie z.B. auf https://www.w3schools.com/sql/

  1. Auf dem RPi sind zwei Datenbanken angelegt. Wir benötigen im Fach EWS (Engineering with Software) nur diejenige mit dem Namen employees. Verschaffen Sie sich einen Überblick über diese Datenbank und die Tabellen. Wie viele Tabellen wurden angelegt?

  2. Welche Attribute haben die Tabellen und von welchem Typ sind sie? Um diese Frage zu beantworten, haben Sie zwei Möglichkeiten. a) Öffnen Sie im Datenbanknavigator die Datenbank durch Doppelklick. Eine Übersicht der Datenbank wird rechts in einem Tab “Eigenschaften” dargestellt. b) Sie klappen im Datenbanknavigator Databases – Employees – Tables auf. Durch Doppelklick auf die einzelnen Tabellen öffnet sich rechts eine Tabellenübersicht. So arbeiten Sie sich durch alle Tabellen.

  3. Als nächstes wollen wir uns die Einträge in den Tabellen ansehen. Dazu dient die SQL-Anweisung SELECT. Nutzen Sie die folgende Webseite und lassen Sie sich alle Departemente anzeigen: https://www.w3schools.com/sql/sql_select.asp Statement:

Um das Statement zu testen, markieren Sie links die employees Datenbank und öffnen über das Menü SQL-Editor ein SQL-Skript (Standarteinstellung F3). Durch Eingeben und Ausführen des Statements werden Ihnen die Resultate des Statements angezeigt.

  1. Nutzen Sie die eingangs erwähnte Quelle aus dem WWW und beantworten Sie damit die folgenden Fragen a) Wie viele Einträge hat die Tabelle employees? (Nutzen Sie SELECT, COUNT) b) Wie viele Mitarbeiter haben den Vornamen ‘Hugo’? (SELECT, COUNT, WHERE) c) Welches ist das höchste Gehalt (SELECT, MAX)
