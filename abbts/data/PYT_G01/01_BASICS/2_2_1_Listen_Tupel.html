2.2.1 Listen & Tupel#


← Übersicht

Listen und Tupel sind sequentielle Datentypen mit einer bestimmten Anzahl von Elementen. Die Elemente können dabei via Indizierung adressiert werden. Der wesentliche Unterschied zwischen einer Liste und einem Tupel besteht in der Veränderbarkeit bzw. Unveränderbarkeit:

  • Liste: veränderbar (mutable), d. h. Elemente können manipuliert werden (überschrieben, gelöscht, hinzugefügt etc.)

  • Tupel: unveränderbar (immutable), d. h. Elemente können nicht manipuliert werden (überschrieben, gelöscht, hinzugefügt etc.)

Alle nachfolgend erläuterten Operationen, die zu keiner Veränderung des Inhalts führen, existieren daher sowohl für Listen als auch für Tupel (z. B. .index(), .count(), etc.). Operationen, welche den Inhalt verändern, sind aufgrund der Unveränderbarkeit von Tupeln nur für Listen definiert (z. B. .pop(), .append(), .insert()).

Die untenstehenden Operationen wollen wir uns anschauen:

  • Elemente einer Liste anhängen

  • Einfügen von Elementen

  • Elemente aus einer Liste entfernen

  • Inhalt einer Liste

  • Index ermitteln

  • Listen sortieren

liste = [100, 3.9, 451, 55, 34]
tupel = (100, 3.9, 451, 55, 34)
print(liste)
print(tupel)

[100, 3.9, 451, 55, 34]
(100, 3.9, 451, 55, 34)

2.2.1.1 Elemente ändern (überschreiben)#

Wie wir im letzten Kapitel gesehen haben, können Elemente einer Liste via Indexierung und Zuweisung überschrieben werden.

liste[1] = 99
print(liste)

[100, 99, 451, 55, 34]

Dies ist bei Tupeln nicht möglich, da diese unveränderbar (immutable) sind. Die folgenden Codezeilen führen daher zu einer Fehlermeldung.

tupel[1] = 99  # nicht erlaubt, da Tupel unveränderbar (immutable)

---------------------------------------------------------------------------
NameError                                 Traceback (most recent call last)
Cell In[1], line 1
----> 1 tupel[1] = 99  # nicht erlaubt, da Tupel unveränderbar (immutable)

NameError: name 'tupel' is not defined

2.2.1.2 Element(e) anhängen (am Ende)#

# am Ende einer Liste ein Element anfügen
liste.append(345.34)
print(liste)

[100, 99, 451, 55, 34, 345.34]

# falls mehrere Element hinzugefügt werden sollen
liste.extend([45, 56, 10.9])  # Parameter muss iterierbar sein, alternativ kann auch +-Operator verwendet werden.
print(liste)

[100, 99, 451, 55, 34, 345.34, 45, 56, 10.9]

Achtung: Falls oben .append anstelle von .extend verwendet worden wäre, wären die drei Elemente [45, 56, 10.9] als Unterliste (= 1 Element) angehängt worden. Somit wäre eine verschachtelte Liste entstanden.

2.2.1.3 Elemente an beliebiger Stelle einfügen#

# einfügen eines Elementes an eine beliebige Stelle innerhalb einer Liste 
# (einfügen am Ende -> append())
liste.insert(2, 'neues Element')
print(liste)

[100, 99, 'neues Element', 451, 55, 34, 345.34, 45, 56, 10.9]

2.2.1.4 Element entnehmen#

# i-tes Element zurückgeben und aus Liste entfernen
letztes_element = liste.pop()  # ohne Parameter wird das letzte Element entfernt
print(letztes_element)
print(liste)

10.9
[100, 99, 'neues Element', 451, 55, 34, 345.34, 45, 56]

i_te_element = liste.pop(2)  # Element mit Index = 2 entfernen
print(i_te_element)
print(liste)

neues Element
[100, 99, 451, 55, 34, 345.34, 45, 56]

# entfernen eines Elementes (Wertes) ohne Kenntnis des entsprechenden Indexes
liste.remove(34)  # es wird nur jeweils ein Element entfernt. Falls Wert mehrfach vorhanden ist, muss remove() mehrmals ausgeführt werden.
print(liste)

[100, 99, 451, 55, 345.34, 45, 56]

2.2.1.5 Zählen / auf Element prüfen#

Will man wissen, wie oft ein bestimmtes Element enthalten ist, kann die Methode .count() nützlich sein. Geht es nur um die Frage, ob ein bestimmtes Element enthalten ist, so ist der in-Operator sehr praktisch (den wir später bei den Schleifen auch wieder antreffen werden).

# zählen von Werten in einer Liste/Tupel
anzahl = liste.count(345.34)
print(anzahl)

# ist entsprechendes Element enthalten?
print(345.34 in liste)

1
True

Der Index eines bestimmten Elementes kann mit der Methode .index() ermittelt werden. Häufig kennt man den Index des gewünschten Elementes ja nicht, sondern nur den Wert des Elementes.

# Position eines Elementes innerhalb einer Liste ermitteln 
# (es wird nur der Index des ersten gefundenen Elements zurückgegeben)
liste.index(55)
# optional können noch Start- und Endindex der Suche übergeben werden, z.B. liste.index(55, 1, 3)

3

2.2.1.6 Sortieren#

Häufig will man sequentielle Datentypen sortieren. Das Listen verändert werden können, weist die Liste eine Methode .sort() auf, welche die Liste in-place sortiert (in-place bedeutet, dass die Elemente in der Liste effektiv umsortiert werden, also keine neue Liste erstellt wird und somit auch nichts zurückgegeben wird). Will man die originale Liste nicht verändern und stattdessen eine neue Liste erzeugen, die sortiert ist, dann verwendet man die Funktion sorted() (keine Methode), welche auch für unveränderbare Datentypen (Tupel, Strings) funktioniert.

print('Die unsortierte Liste: ', liste)
liste.sort()  # sort() liefert nicht die sortierte Liste zurück! Rückgabewert ist None
print('Die sortierte Liste: ', liste)
# Alternative
liste_sortiert = sorted(liste)
print('Die sortierte Liste: ', liste_sortiert)
# die Sortierreihenfolge kann auch invertriert werden
liste.sort(reverse=True)  # oder auch sorted(liste, reverse=True)
print('Die Liste umgekehrt sortiert: ', liste)

Die unsortierte Liste:  [100, 99, 451, 55, 345.34, 45, 56]
Die sortierte Liste:  [45, 55, 56, 99, 100, 345.34, 451]
Die sortierte Liste:  [45, 55, 56, 99, 100, 345.34, 451]
Die Liste umgekehrt sortiert:  [451, 345.34, 100, 99, 56, 55, 45]

2.2.1.7 Mehrfachzuweisungen / Packing & Unpacking#

Python erlaubt auch Mehrfachzuweisungen. Dies ist besonders nützlich im Zusammenhang mit Rückgabewerten von Funktionen etc. Die rechte Seite des Beispiels unten 0, 1 bildet ein Tupel, welches in die numerischen Variablen a und b “aufgeteilt” wird.

a, b = 0, 1  # auch mit Listen möglich
print(a)
print(b)

0
1

erste, *weitere = 34, 44, 6743, 89
print(erste)
print(weitere)

34
[44, 6743, 89]
