1. Einführung#


← Übersicht

Für den praktischen Teil des Moduls werden wir mit der RGB-Matrix der Basislernplattform (BLP) arbeiten. Dieser Aufbau wird für die Aufgaben, wie auch die Modulschlussprüfung verwendet. Anhand verschiedener Aufgabenstellungen soll der Einstieg in Python und die entsprechenden Entwicklungsumgebungen geübt und gefestigt werden.

1.1 RGB-Farbraum#

Da wir die LEDs der RGB-Matrix in verschiedenen Farben leuchten lassen werden, müssen wir uns vorgängig kurz damit befassen, wie wir eine Farbe definieren, da eine Farbe grundsätzlich auf verschiedene Arten repräsentiert werden kann. Eine bei Programmiersprachen (und auch in vielen anderen Bereichen) sehr verbreitete Variante ist die RGB-Repräsentation. Die Grundidee dahinter besteht darin, dass eine Farbe als eine additive Überlagerung von drei Grundfarben (Rot, Grün, Blau) verstanden werden kann. Eine beliebige Farbe kann daher mit drei Komponenten r, g, b beschrieben werden, wobei für die einzelnen Anteile vielfach 8-Bit Werte (0-255) verwendet werden.

Nachfolgend werden einige Standardfarben definiert. Die Farben werden dabei als Listen mit drei Elementen [r, b, g] definiert (alternativ könnten Tupel verwendet werden).

red = [255, 0, 0]
green = [0, 255, 0]
blue = [0, 0, 255]

yellow = [255, 255, 0]
magenta = [255, 0, 255]
cyan = [0, 255, 255]

black = [0, 0, 0]
white = [255, 255, 255]

Natürlich können auch beliebige Zwischenfarben und Intensitäten dargestellt werden. Wir werden feststellen, dass obige Farbdefinitionen für die RGB-Matrix nicht ideal sind, da die LEDs damit sehr hell leuchten.

Anmerkung: In anderen Sprachen/Tools wird das RGB-Triplet häufig auch mit Gleitkommazahlen, d. h. [0-100%, 0-100%, 0-100%] bzw. [0-1, 0-1, 0-1] angegeben bzw. im Internet (HTML/CSS) werden Strings mit Hexadezimalwerten verwendet, z. B. #FF0000 (Rot) etc.

1.2 Einführungsbeispiel RGB-Matrix#

Dieses Einführungsbeispiel zeigt den korrekten Umgang mit dem RGB-Modul auf der Basislernplattform (BLP). Damit die RGB-Matrix korrekt funkioniert, muss der entsprechende COM-Port korrekt eingestellt werden. Hierfür muss die RGB-Matrix an einem USB-Port eingesteckt werden und im Geräte-Manager kann danach der entsprechende COM-Port abgelesen werden.

In diesem Fall ist dies COM5. Der COM-Port muss bei der Initialisierung des RGB-Objektes in Python angegeben werden, wie wir weiter unten sehen werden. Für ein bestimmtes Gerät wird vom Betriebssystem in der Regel immer derselbe COM-Port verwendet, d. h. Sie müssen diesen nicht bei jedem Einstecken nachschauen.

1.2.1 Imports#

Wir werden an anderer Stelle noch im Detail auf das Importieren von externen Paketen bzw. Funktionalität aus anderen Python-Files etc. eingehen. Im Moment genügt es zu wissen, dass wir zur Ansteuerung der RGB-Matrix die Klasse RgbFpga verwenden, die wir aus dem Paket abbts_blp importieren.

Falls Sie das Paket abbts_blp noch nicht installiert haben, können Sie dies mittel pip install abbts_blp (via Terminal) oder direkt in PyCharm → Packages) installieren.

from abbts_blp.rgb import RgbFpga

Wir werden uns später mit der ojektorientierten Programmierung beschäftigen und dann die nachfolgenden Codeteile komplett verstehen. Um etwas auf die RGB-Matrix schreiben zu können, durchlaufen wir im Wesentliches folgende Schritte. Zuerst muss ein Objekt der Klasse RgbFpga erzeugt werden, wir nennen dieses üblicherweise rgb. Dieses Objekt hat ein Attribut rgb_matrix, welches die LED-Matrix repräsentiert (verschachtelte Liste mit 8 x 8 Elementen, entsprechend den 8 Zeilen/Spalten der LED-Matrix). Dieses können wir manipulieren, um ein bestimmtes Muster zu generieren.

  • Objekt erzeugen → rgb = RgbFpga(port='COM5')

  • Matrix manipulieren → Attribut rgb.rgb_matrix verändern etc.

Mit den obigen beiden Schritten wird aber noch nichts an die RGB-Matrix gesendet, d. h. es findet noch keine Kommunikation mit der RGB-Matrix statt. Mit den drei folgenden Methoden wird die Schnittstelle zur RGB-Matrix geöffnet, dann die oben veränderten Daten übermittelt und schliesslich die Schnittstelle wieder geschlossen.

  • Schnittstelle öffnen → Methode rgb.open()

  • Schnittstelle beschreiben → Methode rgb.write()

  • Schnittstelle schliessen → Methode rgb.close()

Im folgenden Beispiel wird dies gezeigt, indem eine einzelne LED in einer bestimmten Farbe eingeschaltet wird.

zeile = 1  # Zeilenindexierung 0..7 (horizontal)
spalte = 1  # Spaltenindexierung 0..7 (vertikal)

rot = 0  # Rotanteil 0..255
gruen = 10  # Grünanteil 0..255
blau = 0  # Blauanteil 0..255

rgb = RgbFpga(port='COM5')  # Erzeugung des rgb-Objektes
rgb.rgb_matrix[zeile][spalte] = [rot, gruen, blau]  # Farbzuweisung

rgb.open()  # Schnittstelle öffnen
rgb.write()  # Schnittstelle schreiben
rgb.close()  # Schnittstelle schliessen
