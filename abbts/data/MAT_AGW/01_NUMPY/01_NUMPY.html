Arrays#


← Modulübersicht

NumPy (Numerical Python) ist eine extrem mächtige Bibliothek für numerische Berechnungen mit Python im technisch-wissenschaftlichen Umfeld und bildet die Basis für viele weitere Pakete des Python-Ecosystems. Als wesentliche Erweiterungen von NumPy gegenüber Basis-Python können genannt werden:

  • leistungsfähiger Datentyp für mehrdimensionale Datenobjekte (Arrays)

  • leistungsfähige Algorithmen für mehrdimensionale Arrayobjekte

  • umfangreiche Sammlung mathematischer Funktionen

Wir werden NumPy schrittweise kennenlernen und über das gesamte Modul hinweg intensiv einsetzen, wobei nach und nach weitere Bibliotheken dazustossen.

Voraussetzungen#

Die Python-Grundlagen sowie der Umgang mit Jupyterlab bzw. PyCharm werden vorausgesetzt und müssen gegebenenfalls selbstständig aufgefrischt werden. Speziell erwähnt seien:

  • Datentypen, v.a. numerische Datentypen und Listen (Erstellung, Indizierung etc.)

  • Kontrollstrukturen (Schleifen, Verzweigungen, Funktionen)

  • OOP: Zugriff auf Attribute & Methoden

  • Arbeiten mit Jupyterlab (Unterschiede zu einer Entwicklungsumgebung wie PyCharm etc.)

  • Debugging / Verwenden der Hilfe & der Dokumentation

Speziell die Indizierung soll hier kurz repetiert werden. Die Indizierung einer einfachen Liste erfolgt bekanntlich mit [], wobei das erste Element mit dem Index 0 angesprochen wird.

liste = [5, 3, 2, 8, 1]
liste[0]  # erstes Element
liste[-1]  # letztes Element
liste[1:4]  # Liste mit zweitem, drittem und viertem Element (Slicing)

Bei verschachtelten Listen kommt pro Verschachtelungsebene ein Index hinzu, wobei dann mit mehreren Klammernpaaren [][] etc. indiziert wird.

liste = [[1, 2], ['a', 'b', 'c']]
liste[0]  # erstes Element der obersten Verschachtelungsebene (= [1, 2])
liste[1]  # zweites Element der obersten Verschachtelungsebene (= ['a', 'b', 'c'])
liste[0][1]  # zweites Element der ersten Unterliste (= 2)

1. Motivation#

Die grundlegenden Python-Datentypen (list, tuple, dict) eignen sich nicht für mathematische Berechnungen mit grossen Datenmengen. NumPy bringt mit dem NumPy-Array (ndarray) einen mächtigen Datentyp mit sich, der das optimierte Handling grosser Datenmengen sowie effiziente Berechnungen ermöglicht.

Im vorliegenden Notebook werden wir die wichtigsten Elemente von NumPy durchgehen, doch es nicht möglich (und auch nicht sinnvoll) alles bis ins letzte Detail anzuschauen. An dieser Stelle soll daher auch auf das Cheatsheet hingewiesen werden, welches das Wichtigste kompakt zusammenfasst. Für weiterführende Informationen können folgende Ressourcen erwähnt werden:

  • NumPy User Guide

  • NumPy Reference

1.1 Import#

NumPy (Paketname numpy) wird üblicherweise mit dem Alias np importiert.

import numpy as np

2. Array (ndarray)#

Der ndarray (nachfolgend einfach Array genannt) ist die zentrale Datenstruktur der NumPy-Bibliothek und kann ein-, zwei- oder auch mehrdimensional sein. Da der Array viele Ähnlichkeiten mit der Liste aufweist, können wir bzgl. Erstellung und Indizierung auf das vorhandene Wissen aufbauen.

Einen eindimensionalen Array (1D-Array) sollte man sich ohne Orientierung (also nicht als Zeilen- oder Spaltenarray) vorstellen, sondern einfach als Anordnung von Elementen (wie eine Liste).

a = np.array([-1.23, 0, 2.5])  # manuelle Erstellung eines Arrays
print(a)
type(a)

[-1.23  0.    2.5 ]

numpy.ndarray

2.1 Erstellung#

Für die Erstellung von Arrays wird meistens eine der folgenden Möglichkeiten angewendet:

  • np.array() (für die manuelle Erzeugung)

  • np.arange() (analog zur Funktion range(), aber nicht auf Ganzzahlen beschränkt)

  • np.linspace() (wie np.arange(), aber mit Sampleanzahl anstelle der Schrittweite)

  • np.logspace() (wie np.linspace(), aber mit logarithmischer Teilung)

a = np.array([-1, 0, 1, 2])  # ähnlich zur Erstellung einer Liste
print(a)
a = np.arange(0, 1, step=0.2) # (Startwert, Endwert (nicht inkl.), Schrittweite)
print(a)
a = np.linspace(0, 5, num=9)  # (Startwert, Endwert (inkl.), Sampleanzahl)
print(a)
a = np.linspace(0, 5, num=9, endpoint=False)  # wie oben, aber diesmal Endwert nicht inkl.
print(a)
a = np.logspace(-2, 3, num=6, base=10.0)  # logarithmische Teilung
print(a)

[0.    0.625 1.25  1.875 2.5   3.125 3.75  4.375 5.   ]
[0.         0.55555556 1.11111111 1.66666667 2.22222222 2.77777778
 3.33333333 3.88888889 4.44444444]
[1.e-02 1.e-01 1.e+00 1.e+01 1.e+02 1.e+03]

Höherdimensionale Arrays können ebenfalls ähnlich wie Listen mit np.array() erstellt werden.

Bei mehrdimensionalen Arrays ist die Orientierung (Zeilen/Spalten) relevant.

Für das Erstellen von Arrays mit Nullen bzw. Einsen gibt es ebenfalls nützliche Funktionen. Daneben gibt es eine Reihe von Funktionen, welche man im Zusammenhang mit der lin. Algebra (Vektor- bzw. Matrizenrechnung) häufig benötigt.

  • np.zeros() bzw. np.ones() (für die Erstellung eines Arrays mit 0’en bzw. 1’en)

  • np.eye() (Einheitsmatrix)

  • np.diag() (Diagonalmatrix)

a = np.zeros(10)  # 1D-Array mit 10 Elementen (alle Elemente = 0)
print(a)
a = np.ones(shape=(3,5))  # 2D-Array mit 3 Zeilen und 5 Spalten (alle Elemente = 1)
print(a)

2.1.1 Import/Export von Textdateien#

Zwei- und höherdimensionale Arrays werden häufig im Zusammenhang mit eingelesenen Daten (Textfiles, Bilder, etc.) verwendet und daher selten mit np.array() manuell erstellt. Für das Arbeiten mit Textdateien sind die folgenden beiden Funktionen zu erwähnen:

  • np.loadtxt() (Einlesen eines Textfiles)

  • np.savetxt() (Abspeichern in Textfile)

Wie immer beim Einlesen von Textfiles ist darauf zu achten, ob Headerzeilen vorhanden sind, die ignoriert werden müssen und welches Zeichen als Trennzeichen verwendet wird. Die obigen Funktionen enthalten dafür entsprechende Parameter (skiprows bzw. delimiter).

a = np.array([[0, 0.5, -1],  # zweidimensionaler Array (manuell erstellt)
              [0.2, 3, 0]])

data = np.loadtxt('data/test.txt')
print(data)

2.2 Elementdatentyp#

Da NumPy für maximale Performance entwickelt wurde, unterscheidet sich der Array dahingehend von der Liste, dass sämtliche Elemente im Array denselben Datentyp aufweisen. Dieser Elementdatentyp kann bei der Erstellung gezielt zugewiesen werden und es steht eine Reihe von spezifischen Datentypen zur Verfügung, wobei die Zahl jeweils für die entsprechende Bitgrösse steht:

  • np.uint8/16/32/64 für vorzeichenlose Ganzzahlen (unsigned integer)

  • np.int8/16/32/64 für vorzeichenbehaftete Ganzzahlen (signed integer)

  • np.float16/32/64/96/128 für Gleitkommazahlen (floating point)

  • np.complex64/128/192/256 für komplexe Zahlen (complex)

Wird der Datentyp nicht explizit angegeben, wählt NumPy automatisch einen (möglichst) passenden Datentyp.

a = np.array([-1, 0, 1, 2])  # automatische Datentyp-Zuweisung (Ganzzahl)
print(a, a.dtype)
a = np.array([0.1, 1, 10])  # automatische Datentyp-Zuweisung (Gleitkommazahl)
print(a, a.dtype)
a = np.array([[-1, 0, 1, 2],
              [0, 1, 3, 1]], dtype=np.float32)  # explizite Datentyp-Zuweisung
print(a, a.dtype)

[-1  0  1  2] int32
[ 0.1  1.  10. ] float64
[[-1.  0.  1.  2.]
 [ 0.  1.  3.  1.]] float32

Der Datentyp kann mit der Methode .astype() geändert werden.

b = a.astype(np.int8)
b

array([[-1,  0,  1,  2],
       [ 0,  1,  3,  1]], dtype=int8)

2.3 Attribute (Eigenschaften)#

Der NumPy-Array kommt mit vielen Attributen, welche nützlich sind, um z. B. Dimension, Grösse, Datentyp etc. abfragen zu können.

  • .ndim (Dimension des Arrays)

  • .shape (Form, d.h. Anzahl Zeilen & Spalten etc.)

  • .size (Gesamtanzahl Elemente)

  • .dtype (Element-Datentyp)

  • .real (Realanteil bei kompl. Zahl)

  • .imag (Imaginäranteil bei kompl. Zahl)

Die Python-Funktion len() gibt bei Arrays jeweils die Länge der ersten Dimension (“Anzahl Zeilen”) zurück. Besonders bei höherdimensionalen Arrays bzw. wenn sich die Arraydimensionen verändern, wird deshalb besser mit .shape gearbeitet.

print(a.ndim)
print(a.shape)
print(a.size)
print(a.dtype)

2
(2, 4)
8
float32

2.4 Array-Methoden#

Neben den oben erwähnten Attributen bietet der NumPy-Array viele nützliche Operationen direkt als Methoden an, was eine elegante/übersichtliche Programmierung ermöglicht.

Nützliche Methoden:

  • .max() bzw. .min() (Maximum bzw. Minimum des Arrays)

  • .argmax() bzw..argmin() (Index des Maximums bzw. Minimums)

  • .mean(), .std(), .var() (Mittelwert, Standardabweichung, Varianz)

  • .sum() bzw. .cumsum() (Summe bzw. kumulierte Summe)

  • .prod() bzw. .cumprod() (Produkt bzw. kumuliertes Produkt)

  • .round() (Runden)

  • .sort() (Sortierung)

  • .transpose() (Transponierung)

Viele der obigen Methoden existieren auch als allg. Funktionen in NumPy. Das Maximum eines Arrays kann z. B. entweder via Methode a.max() oder via Funktion np.max(a) bestimmt werden.

Bei Funktionen wie z. B. .sum() oder .max(), bei denen die Anwendung auf einen höherdim. Array nicht eindeutig ist, kann mit dem Parameter axis angegeben werden, ob die Operation zeilen- bzw. spaltenweise durchgeführt werden soll. axis = 0 (Summe für jede Spalte) axis = 1 (Summe für jede Zeile) axis = None (Default: Summe über alles)

print(a.mean())
print(a.sum())
print(a.sum(axis=0))
print(a.sum(axis=1))

0.875
7.0
[-1.  1.  4.  3.]
[2. 5.]

3. Form/Gestalt eines NumPy-Arrays#

Sehr wichtig im Zusammenhang mit Arrays (besonders mit höherdimensionalen) ist, dass man sich immer im Klaren über die Dimension und die Form/Gestalt (shape) des Arrays ist, da das Debugging sonst mühsam ausfallen kann. Dies ist v. a. in Bezug auf die Indizierung sehr wichtig. Wir werden primär mit 1D- und 2D-Arrays zu tun haben, es sind aber auch 3- und höherdimensionale Arrays möglich.

3.1 Formänderung#

Die Array-Methode .reshape() ermöglicht es, die Dimension eines Arrays zu ändern, d.h. z. B. einen 1D-Array mit 6 Elementen in einen 2D-Array mit z. B. 2 Zeilen & 3 Spalten zu verwandeln.

a = np.arange(3, 0, -0.25)  # 1D-Array mit 12 Elementen
print(a)
b = a.reshape((3,4))  # 2D-Array mit 3 Zeilen und 4 Spalten (= 12 Elemente)
print(b)

[3.   2.75 2.5  2.25 2.   1.75 1.5  1.25 1.   0.75 0.5  0.25]
[[3.   2.75 2.5  2.25]
 [2.   1.75 1.5  1.25]
 [1.   0.75 0.5  0.25]]

3.2 Indizierung und Slicing#

Wie bei Listen erfolgt das Indizieren/Slicing mit [].

# 1D-Array
print(a[0])  # erstes Element (Index 0)
print(a[-1])  # letztes Element
print(a[4:9])  # Slicing

3.0
0.25
[2.   1.75 1.5  1.25 1.  ]

Für höherdimensionale Arrays ist die Indizierung bedeutend intuitiver als bei Listen, da nicht mehrere Klammernpaare verwendet werden müssen, sondern die Indizes mit , getrennt werden können.

# 2D-Array
print(b[0,0])  # Element in erster Zeile / erster Spalte
print(b[1,:])  # alle Elemente der zweiten Zeile (alternativ b[1])
print(b[:,2])  # alle Elemente der dritten Spalte

3.0
[2.   1.75 1.5  1.25]
[2.5 1.5 0.5]

Besonders erwähnenswert im Zusammenhang mit der Indizierung ist die boolsche Indizierung (boolean indexing), die sehr praktisch ist, um einen Array anhand seiner Elemente (Werte) zu filtern, d.h. nur bestimmte Elemente zu identifizieren bzw. zu überschreiben etc.

print(a[a>2])  # alle Elemente aus a, die grösser als 2 sind
print(a[(a>1)&(a<2)])  # alle Elemente aus a, die grösser als 1 und kleiner als 2 sind
a[(a>=2.5)|(a<=0.5)] = 0  # alle Elmente aus a, die grösser/gleich 2.5 oder kleiner/gleich 0.5 sind auf 0 setzen
a

[3.   2.75 2.5  2.25]
[1.75 1.5  1.25]

array([0.  , 0.  , 0.  , 2.25, 2.  , 1.75, 1.5 , 1.25, 1.  , 0.75, 0.  ,
       0.  ])

3.3 Editieren/Kopieren#

Elemente in Arrays können verändert werden, wie wir dies von Listen kennen. Wie bei Listen muss beachtet werden, dass eine Variablenzuweisung eines Arrays a zur Variable b, also b = a, keine Kopie aller Arrayelemente vornimmt, sondern lediglich die Referenz auf die Arrayelemente kopiert. b “enthält” dann dieselben Elemente wie a und wenn ein Element in a bzw. b verändert wird, so zeigt sich diese Änderung auch in b bzw. a.

a = np.array([1, 2, 3])
b = a  # keine Kopie, nur Zuweisung (Referenz)
a[1] = -99  # auch a wird verändert
print(a, b)

a = np.array([1, 2, 3])
b = a.copy()  # Kopie
b[1] = -99  # a ist nicht betroffen
print(a, b)

[  1 -99   3] [  1 -99   3]
[1 2 3] [  1 -99   3]

4. Numerische Berechnungen#

NumPy bringt ebenfalls alle relevanten Konstanten und mathematischen Funktionen mit sich, die man für technisch-wissenschaftliche Berechnungen benötigt. Der Import des math-Moduls erübrigt sich, wenn mit NumPy gearbeitet wird.

4.1 Konstanten#

Nützliche Konstanten:

  • np.pi (Kreiszahl \(\pi\))

  • np.e (Eulersche Zahl \(e\))

  • np.inf (unendlich, z. B. bei Division durch 0)

  • np.nan (undefiniert, not a number)

4.2 Funktionen#

Eine Auswahl von mathematischen Funktionen:

  • np.sqrt(), np.square(), etc.

  • np.sin(), np.cos(), np.tan(), etc.

  • np.arcsin(), np.arccos(), np.arctan() bzw. np.arctan2(), etc.

  • np.exp(), np.log(), np.log10(), etc.

Nützliche Umrechnungsfunktionen bzw. Hilfsfunktionen:

  • np.absolute(), np.sign(), etc.

  • np.degrees() bzw. np.rad2deg(), np.radians() bzw. np.deg2rad(), etc.

  • np.angle(), np.real(), np.imag(), etc.

Funktionen zum Runden:

  • np.round(), np.floor(), np.ceil() etc.

Ein grosser Vorteil von NumPy liegt in der Einfachheit, mit der mathematische Berechnungen mit Arrays durchgeführt werden können. Am Beispiel der folgenden Formel zur Berechnung der Standardabweichung wollen wir dies illustrieren. (Würde man mit einer Liste arbeiten, könnte die Berechnung nicht so direkt umgesetzt werden und es wäre eine Schleife oder z. B. die map-Funktion nötig.)

\[s = \sqrt{ \frac{1}{n-1} \sum_{i=0}^{n} (x_i-\bar x)^2 }\]
# Die obige Formel mit NumPy
n = 100
x = np.arange(n)

s = np.sqrt(1/(n-1) * np.sum(np.square(x-x.mean())))
s

29.01149197588202
